01
Overview

OmniMed AI is a unified multimodal diagnostic platform integrating three distinct medical specialties — Neurology, Orthopaedics, and Dermatology — into a single clinical interface. Most medical AI research assumes access to enterprise-grade GPUs (14 GB+ VRAM). OmniMed's core innovation is its Hardware-Agnostic Architecture, enabling state-of-the-art deep learning models to run on consumer-grade hardware with only 6 GB VRAM, making advanced diagnostics accessible to small clinics and resource-limited settings.

02
Architecture

The core innovation is the Singleton Model-Loading Mechanism. Instead of loading all three models simultaneously — which would exceed 6 GB VRAM — the system treats GPU memory as a shared resource. Only one model occupies VRAM at a time. On module switch, the active model is fully unloaded and torch.cuda.empty_cache() is triggered to purge residual allocations before the next architecture is loaded. The frontend (Next.js + Tailwind CSS) provides a seamless dark-mode clinical dashboard, while the Python/PyTorch REST backend handles all inference and image pre-processing (auto-resize to model-specific input, e.g. 224×224 for ResNet).

03
Medical Modules

Neurology (2D U-Net): Brain MRI abnormality segmentation. 3D NIfTI volumes are decomposed into 2D slices for lesion and tumour detection within the 6 GB constraint. Orthopaedics (ResNet-18): Binary classification (Normal vs. Abnormal) on musculoskeletal X-rays using the MURA dataset, providing a second opinion in high-throughput trauma centres. Dermatology (EfficientNet-B0): Skin lesion classification (Malignant vs. Benign). EfficientNet's compact footprint is ideal for the shared-VRAM model — high accuracy, minimal memory overhead.

04
Impact

OmniMed AI directly challenges the assumption that medical AI requires cloud compute or enterprise GPUs. By reducing peak VRAM by over 57%, the platform runs on standard laptops found in small clinics and rural healthcare centres. The unified dashboard eliminates the need to switch between three separate diagnostic tools, reducing cognitive load for clinicians. The Singleton Loader is plug-and-play: adding a Cardiology or Pulmonology module requires no hardware upgrade. Limitations include current reliance on 2D slice extraction for MRI (full 3D inference is a future target) and dataset scope for orthopaedic classification.

05
Conclusion

OmniMed AI demonstrates that multi-specialty medical AI diagnostics can be delivered on consumer-grade hardware through intelligent GPU memory management. The Singleton Loader architecture is a validated, extensible framework for real-world clinical deployment. Future work will target full 3D MRI volumetric inference, expanded specialty modules (Cardiology, Pulmonology), and cloud-optional hybrid deployment.
